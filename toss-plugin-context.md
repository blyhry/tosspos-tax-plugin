# 토스포스 플러그인 아이디어 탐색 — 컨텍스트 파일

> 마지막 업데이트: 2026-03-16 (2차 세션 반영)

---

## 1. 회사 배경 (iShopCare)

- **업종**: 결제 VAN 대리점 (카드 단말기 유통·설치)
- **주요 자산**:
  - 결제 VAN 데이터 (카드 트랜잭션)
  - 사장님과 직접 접점이 있는 유통망
  - 토스포스 단말기 설치 → 토스플레이스 API 접근 가능
- **New Biz (진행 중)**:
  - 배달: 배달 플랫폼 도입 절차 도와주는 수준 (데이터 없음, 재판매)
  - 인터넷: 통신사 인터넷 재판매 (데이터 없음)
- **목표**: 토스포스 플러그인(간단한 것)을 만들어 락인 효과 or 독립 수익 창출

## 2. 1차 기획안 (원문 보존)

### 핵심 가설
> 가맹점 절세에 지불한 비용보다 더욱 도움을 주는 플러그인이라면 비용을 지불해서라도 쓸 것이다.
> 또는 아이샵케어에서 납품한 곳에만 설치해준다면 아이샵케어 락인 효과가 있을 것이다.

### 배경 논리
- 사장님에게 가장 중요한 건 돈 (더 벌거나, 덜 나가거나)
- 사장님 절세는 안내 콘텐츠만 있을 뿐, 데이터 기반 자동 계산 + 액션 안내 서비스 없음
- 직장인 연말정산처럼, 사장님도 절세 관심 많지만 실행 도구가 없음

### 타깃
- **간편장부 대상자**: 업종별 매출 기준 이하 (예: 음식점 직전년도 1억 5천 이하)
- **복식부기 의무자**: 업종별 매출 기준 초과

### 솔루션 내용
**① 간편장부 자동 생성**
- 간편장부 대상자: 매출 데이터 → 국세청 형식 자동 변환, 지출 항목 직접 입력 기능, 제출 가능 파일 제공
- 복식부기 의무자: 매출 분개 4종 자동 생성 (카드매출 발생 / 정산 수령 / 현금영수증 / 취소)

**② 신용카드 등 발행 세액공제 트래킹**
- 카드·현금영수증 매출 × 1.3% 자동 계산
- 연간 한도(1,000만원) 대비 현재 공제액 시각화
- 부가세 신고 시 홈택스 입력 위치 안내 + 반기 알림

**③ 팁 제공 영역 (자동화 불가, 정보 제공)**
- 차량 부가세 공제 (경차, 화물차, 9인 이상 승합차)
- 임차료·관리비 공제 (분기별 세금계산서)
- 의제매입세액공제 (음식점 등)

### 포지셔닝
- 토스: "당신의 매출이 이렇다" → 단순 데이터 조회
- 플러그인: "지금까지 세금 NN만원 아꼈고, 연간 한도 NN만원 남았음" → 행동 가능한 절세 계산

### BM
- 아이샵케어 도입 가맹점에 무료 제공 → 락인 효과
- 데이터·장부 누적 → 이탈 시 손실 → 자연스러운 락인
- 제휴 세무사 또는 토스페이먼츠 연계 수수료 쉐어

### 1차 기획의 문제 가설 (H1, H2)
- **H1**: 사장님은 자신이 절세 혜택을 놓치고 있다는 사실을 모른다
  - 검증: 기존 가맹점 10곳 인터뷰
  - 성립 조건: 7곳 이상 "몰랐다" 또는 "알았는데 신청 안 했다"
- **H2**: 알아도 어떻게 해야 할지 몰라서 포기한다
  - 검증: 알고 있는 가맹점이 홈택스 스스로 진행한 경험 확인
  - 성립 조건: 자력 완료율 30% 미만

### 1차 기획의 우려점 (탐색 중 도출)
1. 국세청 간편장부 프로그램이 생각보다 쓰기 편함 → 차별점 약화
2. 매출 데이터만으로는 자동화 범위가 좁음 → 전체적 메리트 부족
3. 락인 효과가 충분히 강한가? 절세 데이터 하나로 단말기 유지할 사장님이 얼마나 될까

## 3. 탐색한 아이디어와 결론

### ❌ 간편장부 자동화
- **아이디어**: 토스포스 매출 → 간편장부 파일 자동 생성 → 기장신고 유도
- **왜 막혔나**:
  - 국세청 간편장부 프로그램이 생각보다 쓰기 어렵지 않음
  - 매출 데이터만 있고 지출 데이터 없음 → 실질 절세 효과 미미
  - "결국 세무사 쓰는 게 낫다"는 결론

### ❌ 세무사 연계 플랫폼
- **아이디어**: 매출 데이터 정제 → 세무사에게 넘겨 자료 수집 비용 절감
- **왜 막혔나**:
  - 세무사가 외부 툴 데이터를 신뢰할지 불확실
  - 신뢰 구축에 시간/비용 필요
  - 파트너 세무사 확보 전까지 가치 증명 어려움

### ❌ 통합 매출 대시보드 (배달+결제)
- **아이디어**: VAN + 배달 + 인터넷 데이터 통합 → 사장님 원스톱 뷰
- **왜 막혔나**:
  - 배달·인터넷은 재판매 수준 → 데이터 없음
  - 데이터 모트가 실제로는 존재하지 않음

### ❌ 세금 절세 (의제매입세액공제, 통합고용세액공제 등)
- **아이디어**: 매출 데이터 + 업종으로 놓친 세금 혜택 알림
- **왜 막혔나**:
  - 의제매입세액공제는 재료비(지출) 데이터 필요
  - 통합고용세액공제는 직원 정보 필요
  - 결국 지출/비용 데이터 없이는 대부분 불가

### 🟡 정부 지원금 알림 (미검증)
- **아이디어**: 매출+업종+사업연차 → 신청 가능한 소상공인 지원금 매칭
- **장점**: 지출 데이터 불필요, 사장님 입장에서 "공짜 돈" 느낌
- **단점**: 수익 모델 약함, 지원금 DB 관리 부담, 수시 변경
- **결론**: 락인/참여율 도구로는 유효, 단독 수익화는 어려움

### 🟡 보험 추천·연계 (미검증)
- **아이디어**: 매출+업종 → 맞춤 사업자 보험 추천 → 가입 연결 → 수수료
- **장점**: 명확한 수익 구조(보험대리점 수수료), 소상공인 보험 가입률 낮음
- **단점**: 보험대리점 라이선스 필요, 카카오·토스 이미 진입
- **결론**: 라이선스 취득 가능 여부가 핵심 변수

### 💡 현재 유력 방향
- **정부 지원금 알림 (참여율)** + **보험 연계 (수익화)** 조합
- 지원금 알림으로 매일 플러그인 열게 만들고, 그 안에서 보험 자연스럽게 제안

## 5. 다음 단계: 사장님 인터뷰

**아직 검증 안 된 핵심 가정**
- 사장님이 정부 지원금에 관심이 있는가?
- 보험을 POS 앱에서 가입하는 것에 거부감이 없는가?
- 지금 실제로 제일 불편한 게 뭔가?

**인터뷰 질문 (제품 얘기 꺼내지 말고)**

돈 관련:
- 매달 고정으로 나가는 돈이 뭐가 있어요? (카드수수료, 배달수수료, 보험, 임대료...)
- 그중 "이거 왜 내는지 모르겠다" 싶은 게 있어요?
- 세금 신고는 어떻게 해요? 세무사요, 직접이요?

불편함 관련:
- 가게 운영하면서 제일 귀찮은 게 뭐예요?
- 지금 쓰고 있는 앱이나 서비스가 있어요?
- 토스포스에 없어서 아쉬운 기능 있어요?

행동 관련:
- 정부 지원금 신청해본 적 있어요?
- 새로운 서비스 쓸 때 어떻게 알고 써요?

## 6. 2차 세션 추가 탐색 (2026-03-16)

### 확인된 사항
- 보험대리점 라이선스 없음 → 보험 연계 아이디어 폐기
- 정부 지원금 + 보험 조합도 임팩트 약하다고 판단 → 폐기
- 결론: 1차 기획(가맹점 절세)이 여전히 가장 매력적

### 지출 데이터 수집 가능성 탐색
**문제**: 매출 데이터만 있고 지출 데이터 없어서 절세 자동화 범위가 좁음

**발견한 루트 — 홈택스 연동**
홈택스에 사장님 지출 데이터가 이미 존재:
- 사업용 신용카드 매입 내역
- 전자세금계산서 수신 내역 (임차료, 재료비 등)
- 현금영수증 매입 내역

사장님 공동인증서 or 간편인증 동의 1회 → 데이터 수집 가능
삼쩜삼, 자비스, 비즈넵 등이 유사 방식으로 운영 중 (추정, 미확인)

**iShopCare 차별점**
| | 삼쩜삼/자비스 | iShopCare 플러그인 |
|--|------------|-----------------|
| 매출 데이터 | 홈택스 스크래핑 (간접) | 토스포스 실시간 (직접) ✅ |
| 지출 데이터 | 홈택스 스크래핑 | 홈택스 스크래핑 (동일) |
| POS 연동 | X | ✅ 차별점 |

→ 매출(토스포스 실시간) + 지출(홈택스 연동) = 삼쩜삼보다 완성도 높은 장부 자동화 가능

**현실적 장벽**
- 홈택스 공식 API 범위 제한 → 스크래핑 방식 현실적 (법적 그레이존)
- 인증서 관리 UX 복잡
- 삼쩜삼과 직접 경쟁 구도

**미결**: 삼쩜삼과의 경쟁이 현실적인 싸움인지 판단 필요
